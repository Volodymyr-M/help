# リソースビュー

チーム全体の作業負荷配分を確認できます。配分ビューでは誰がいつ何をしているかを表示し、チームプランナーではドラッグして再割り当てでき、リソースグラフではキャパシティを一目で把握できます。

## リソース配分

**Resource Usage**ビューでは、すべてのリソースの割り当てと、各稼働リソースがタイムライン上の各期間に実施する[作業量](/ja/building-schedule/task-properties/index.md#作業量)を確認できます。

ガントチャートと同様に、タイムラインをズームイン・ズームアウトして、より詳細またはより広範なビューを表示できます。

リソースが指定された期間にカレンダーが許容する以上の作業を行わなければならない場合、対応する**Work**が赤色でハイライト表示されます。

Resource Usageビューでは、割り当てレベルの作業セルをダブルクリックして作業配分を直接編集できます。作業セルを編集すると、割り当ての作業量配分は自動的に「Contoured」に変更され、カスタムの日ごとの作業データが保存されます。

## タスク配分

**Task Usage**ビューはResource Usageの逆で、作業配分のタスク中心のビューを表示します。各タスクがトップレベルの行として表示され、そのリソース割り当てがインデントされてその下に表示されます。

左パネルにはタスクと割り当ての詳細が、右パネルには期間にわたる作業量の値が表示されます。タスクレベルの作業量の値は太字で表示されます。過負荷の割り当て作業量の値は赤色でハイライト表示されます。

## チームプランナー

**Team Planner**ビューは、各行が稼働リソースを表すリソース中心のスケジューリングビューです。タスクバーは共有のタイムスケール上で各リソースの行に配置されます。

主な機能：

- **水平にドラッグ**してタスクを再スケジュール — タスクに新しい位置でStart No Earlier Than制約が設定されます
- **垂直にドラッグ**してタスクを別のリソースに再割り当て
- **斜めにドラッグ**して再スケジュールと再割り当てを1回の操作で実行
- **Unassigned Tasks**セクションとの間でドラッグして、リソースの割り当てを追加または削除
- 過負荷のリソースは左パネルに赤色のテキストで表示
- 100%完了したタスクはドラッグできません

## リソースグラフ

**Resource Graph**ビューは、リソースごとの作業負荷配分のヒストグラムを時間経過とともに表示します。一度に1つの稼働リソースが表示され、各期間に対して垂直バーが描画されます。左パネルの矢印ボタンでリソース間を移動します。

7種類のグラフタイプが利用可能です：

| グラフタイプ | ��明 |
|------------|-------------|
| **Peak Units** | 期間あたりのリソースキャパシティ使用率（デフォルト） |
| **Work** | 期間あたりの作業時間 |
| **Overallocation** | キャパシティを超えた超過時間のみ |
| **Percent Allocation** | Peak Unitsと同じくパーセンテージ表示 |
| **Remaining Availability** | 未使用キャパシティ（時間） |
| **Work Availability** | カレンダー上の総キャパシティ（時間） |
| **Unit Availability** | 有効最大単位パーセンテージ |

青色のバーはキャパシティ内の作業を、赤色のバーはキャパシティを超えた過負荷部分を表します。
