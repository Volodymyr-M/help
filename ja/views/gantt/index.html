ガントチャート #

ガントチャートはプロジェクトのタイムラインです。平準化の調整、進捗ライン、ベースライン設定後のスケジュールの変化を確認できます。

利用可能なビュー #

Inganttは、ナビゲーションドロワーまたはViewメニューからアクセスできる、プロジェクト作業用の複数のビューを提供します。すべてのビューが完全に機能し、どのビューでもタスクに対して利用可能なすべての操作を実行できます。

タスクビュー：

  • Tasks — タスクリストとガントチャート
  • Tracking Gantt
  • Task Board
  • Network Diagram
  • Calendar View
  • Timeline

リソースビュー：

  • Resource Usage
  • Task Usage
  • Team Planner
  • Resource Graph

Tasksビュー #

Tasksビューは、タスクリストとガントチャート（分割ビュー）を組み合わせたメインビューです。View > Panels in Tasksサブメニューで、表示するパネルを設定できます。タスクリストとガントチャートはそれぞれ独立して切り替え可能です。

タスクインスペクター #

Task Inspectorは、選択したタスクの詳細を表示するサイドパネルです。スケジューリング要因（タスクの日付を決定する要素）、一般プロパティ、リソース、先行タスク、コストなどが含まれます。タスクインスペクターはツールバーから切り替えます。

インスペクター上部のScheduling Factorsセクションには、タスクのスケジュール日付を決定する要素が表示されます：ドライビング先行タスク（太字で「Driving」バッジ付き）、非ドライビング先行タスク（相対的なフロート付き）、制約、平準化遅延、カレンダー、余裕の値。クリティカルタスクには「Critical」バッジが表示されます。

平準化ガント #

プロジェクトに自動平準化が適用されると、ガントチャート領域にLeveling Ganttトグルボタンが表示されます。

有効にすると、ガントチャートに各タスクの平準化前の位置（自動平準化前のタスクの位置）に緑色のバーが表示されます。標準のタスクバーは現在の平準化後の位置に残ります。これにより、元のスケジュールと平準化後のスケジュールを視覚的に比較し、各タスクがどれだけ遅延されたかを確認できます。

トグルがオフの場合、標準のタスクバーのみが表示されます。

平準化ガントトグルは平準化データが存在する場合にのみ表示されます。平準化をクリアすると自動的に非表示になります。平準化データが既に含まれているプロジェクトファイルを開いた場合、トグルは利用可能ですがオフの状態で開始されます。

進捗ライン #

有効にすると、ガントチャートに進捗ラインが表示されます。これは、ステータス日付に対してタスクが遅れているか進んでいるかを視覚的に示すジグザグ線です。遅れているタスクでは線が左にピーク、進んでいるタスクでは右にピーク、予定通りのタスクでは線がまっすぐになります。

進捗ラインはガントチャートのフローティングツールバーボタンまたはViewメニューから切り替えます。有効な場合、PDFおよび印刷出力にも進捗ラインが含まれます。
