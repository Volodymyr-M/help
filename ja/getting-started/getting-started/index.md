# はじめに

ここから最初のプロジェクトスケジュールを作成しましょう。わずか数分で、タスクをタイムラインに配置し、相互にリンクし、リソースを割り当てることができます。プロジェクトがいつ完了する見込みかを確認するのに十分な内容です。

> モバイルデバイスでは、クリックの代わりにタップ、ダブルクリックの代わりにダブルタップ、右クリックの代わりに長押しを使用してください。

## 基本的なスケジュールの作成

2つのタスクがあり、1つが終わったら次に取りかかる、小規模な住宅リフォームプロジェクトを例にしましょう。

**Create new**をクリックすると、左側にタスクリスト、右側にガントチャートがある分割画面が表示されます。右下の大きな**+**ボタンをクリックして、タスクを順番に追加します。**Add Task**ダイアログで、各タスクの**Name**と予測される**Duration**を指定します。

![タスクの追加](/images/getting-started/add_task.png)

左側のリストで両方のタスクを選択し、ツールバーの**Link tasks**ボタンをクリックします。これでタスクがリンクされ、ガントチャートが更新されます。

- AndroidとiOSで複数のタスクを選択するには、リストまたはガントチャートで各タスクをタップする際に少し長く指を押さえてください。
- その他のプラットフォームでは、キーボードの`Ctrl`または`Shift`を押しながら、リストまたはガントチャートでタスクをクリックしてください。

![リンク](/images/getting-started/link.png)

タスクがリンクされたので、最初のタスクの期間を変更してみましょう。リストまたはガントチャートでダブルクリックし、**Task Properties**ダイアログで**Duration**フィールドを編集します。この変更が2番目のタスクをシフトさせ、スケジュールにどう影響するかを確認できます。

![ガントチャート](/images/getting-started/gantt.png)

## プロジェクト名と開始日の設定

プロジェクトの開始日がInganttのデフォルトと異なる場合があります。

ナビゲーションドロワーから、またはWebおよびWindowsではヘッダーのプロジェクト名をクリックして、**Project Properties**を開きます。

**Project Start Date**フィールドで新しいプロジェクト開始日を設定し、ついでにプロジェクトの**Name**も変更しましょう。

![プロジェクト名](/images/getting-started/project_name.png)

> 基本的なスケジュールのシンプルさで、ガントチャートを見ればプロジェクトがいつ終了する見込みかがすでにわかります。

## リソースの追加と割り当て

プロジェクトの各タスクをそれぞれ別の担当者が行う場合を考えましょう。

ナビゲーションドロワーを開き、**Resources**ビューに切り替えます。

![ナビゲーション](/images/getting-started/nav.png)

大きな**+**ボタンをクリックして**Add Resource**ダイアログを開き、リソースの**Name**を入力して保存します。同じ手順を繰り返して、もう1つの稼働リソースを追加します。

![リソースの追加](/images/getting-started/add_resource.png)

![リソース](/images/getting-started/resources.png)

ナビゲーションドロワーを使って**Tasks**ビューに戻ります。リストまたはガントチャートで最初のタスクをダブルクリックして**Task Properties**ダイアログを開きます。

**Resources**タブに切り替え、最初のリソースにチェックを入れてこのタスクに割り当て、保存します。

![割り当て](/images/getting-started/assignments.png)

2番目のタスクと2番目のリソースについても同じ手順を繰り返します。

![リソース付きガントチャート](/images/getting-started/resources_gantt.png)

## 休暇の設定

プロジェクトの期間中に、タイムラインに影響する予定がリソースに入る場合があります。

最初のタスクを担当するリソースが、そのタスクの期間中に1日休みを取る必要があるとしましょう。

これを設定するには、ナビゲーションドロワーで**Resources**ビューに移動します。

リソースをダブルクリックして**Resource Properties**を表示します。**Calendar**タブに移動し、**Add exception**ボタンをクリックします。プロジェクト期間中の日付を1つ選び、稼働時間のリストを空のままにして保存します。

![休日](/images/getting-started/day_off.png)

ナビゲーションドロワーで**Tasks**に戻り、スケジュールがどう変わったかを確認します。2番目のタスクは最初のタスクへの依存関係により、シフトしています。

![休暇](/images/getting-started/vacation.png)

> 休暇だけでなく、リソースのスケジュールに対するその他の変更も設定できます。たとえば、例外に稼働時間を追加することで、その特定の日にリソースが特定の時間帯のみ稼働することを指定できます。

## 祝日の設定

プロジェクト期間中に祝日があり、その日は誰も作業しないとしましょう。

休暇は各リソースのカレンダーで設定します。一方、全員に影響する休日はプロジェクトのカレンダーで設定します。デフォルトのプロジェクトカレンダーは**Standard**と呼ばれ、週末は作業しないことがすでに設定されています。

祝日を設定するには、ナビゲーションドロワーで**Calendars**ビューに移動します。**Standard**をダブルクリックして**Calendar Properties**を表示します。**Add exception**ボタンをクリックします。プロジェクト期間中の日付を1つ選び、稼働時間のリストを空のままにして保存します。

![祝日](/images/getting-started/holiday.png)

ナビゲーションドロワーで**Tasks**ビューに戻り、プロジェクトがどう変わったかを確認します。タスクがシフトし、祝日はガントチャートで週末と同様に表示されます。

![最終結果](/images/getting-started/final.png)

> 同じ方法で、祝日だけでなく全員のスケジュールに対するその他の変更も設定できます。たとえば、例外に稼働時間を追加することで、その特定の日に全員が特定の時間帯のみ稼働することを指定できます。
>
> また、**Calendar Properties**の各曜日タブも確認できます。チーム全体が金曜日に早退したり、土曜日に作業したりする場合は、対応するタブで設定できます。
