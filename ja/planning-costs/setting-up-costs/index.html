コストの設定 #

タスクに固定費を、リソースに単価を設定しましょう。Inganttはこれらをスケジュールと組み合わせて、各タスクとプロジェクト全体のコストを計算します。

タスクの固定費 #

リソースに関係なくタスクにコストを設定するには、Task PropertiesダイアログのCost and EVタブにあるFixed Costフィールドに入力します。たとえば、タスクの総コスト見積もりが既に提供されている場合や、リソースコスト以外にタスクに追加コストがある場合に使用します。

同じ固定費を複数のタスクに割り当てる必要がある場合は、コストリソースを作成してそれらのタスクに割り当てることを検討してください。将来コストが変更された場合、1か所を更新するだけで済みます。

稼働リソースのコスト #

Workリソースはカレンダーを持つ唯一のタイプであるため、コストは時間、日、週、月の単位で指定します（1日の時間数、1週間の日数、1か月の日数を参照）。

たとえば、Standard Rateが1時間あたり$100の稼働リソースを、Durationが5時間のタスクに割り当てると、タスクのCostに$500が加算されます。

稼働リソースは、通常のカレンダー時間を超える作業に対するOvertime Rateもサポートしています。総コストは次のように計算されます：

コスト = 通常作業量 × 標準単価 + 残業作業量 × 残業単価 + 使用単価

稼働リソースをタスクに割り当てる際、Unitsの値をデフォルトとは異なる数値に指定できます。これはコスト計算に影響します。たとえば、Unitsが50%の場合、タスクにおけるリソースの計算コストは100%の場合の半分になります。

資材リソースのコスト #

Materialリソースの場合、コストは単位の定義に基づいて指定します。重量の単位（ポンド、キログラム、トンなど）、長さや体積の単位（フィート、メートル、マイル、ガロン、リットルなど）、その他の単位（コンテナ、箱、個、平方フィートなど）が可能です。Resource PropertiesダイアログでMaterial Labelを設定して単位を説明できます。

資材リソースは2つの消費モードに対応しています：

  • Fixed（デフォルト） — タスクの期間に関係なく固定数量（例：セメント5トン）
  • Variable — タスクの期間に応じてスケールする時間あたりの割合（例：燃料1日あたり10リットル）

たとえば、プロジェクトで燃料を使用する場合、「燃料」の資材リソースを追加し、Resource Propertiesダイアログでガロンあたりのコストを指定できます。次に、リソースをタスクに割り当てる際に、ガロン数をUnitsとして指定すると、Inganttが燃料の計算コストをタスクのCostに加算します。

コストリソース #

Costリソースは、複数のタスクに割り当てたい固定費用です。コストリソースは割り当てごとのコスト値をサポートしています。同じコストリソースでも、異なるタスクの割り当てで異なる金額を設定できます（例：タスクAの「出張費」=$800、タスクBでは$1,200）。

通常の測定単位では指定しない固定費用（設置費など）にこのタイプを使用してください。同じ理由で複数のタスクに同じ固定費がある場合、各タスクのFixed Costフィールドに入力する代わりに、コストリソースを作成してそれらすべてのタスクに割り当ててください。

単価テーブル #

各稼働リソースおよび資材リソースは最大5つのコスト単価テーブル（AからE）をサポートしており、各テーブルには異なる日付に有効になる複数の行があります。これにより、時間の経過に伴う単価変更のモデリングや、異なるタイプの作業に対する異なる単価セットの管理が可能です。

単価テーブルはResource PropertiesダイアログのCostsタブで設定します。リソースをタスクに割り当てる際、Edit Resource Assignmentダイアログでどの単価テーブルを使用するかを選択できます。

使用単価 #

リソースはCost Per Useフィールドをサポートしています。これは、作業量に関係なく、リソースがタスクに割り当てられるたびに課金される一律料金です。稼働リソースの場合、Cost Per Useは割り当て単位で乗算されます（例：200%の単位は使用単価が2回課金されることを意味します）。Cost Per Useは常にタスクの開始時に発生します。

コスト発生 #

Cost Accrual設定は、作業の進行に伴いコストが実績コストとしていつ認識されるかを制御します：

発生方法 コスト認識のタイミング
Start 少しでも進捗があった時点で全額コスト（達成率 > 0%）
Prorated 達成率に比例（例：50%完了 = コストの50%）
End タスクが100%完了した時点でのみ全額コスト

リソースにはResource PropertiesダイアログのCostsタブにCost Accrual設定があります。タスクにはTask PropertiesダイアログのCost and EVタブにFixed Cost Accrual設定があります。プロジェクトにはProject PropertiesダイアログのOtherタブにDefault Fixed Cost Accrual設定があります。
