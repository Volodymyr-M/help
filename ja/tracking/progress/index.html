進捗の追跡 #

作業が開始されたら、各タスクの**% Complete**を更新して、実際の進捗が計画とどのように比較されるかを追跡します。Update Projectを使用して、一括で進捗を設定できます。進捗が記録されると、Inganttは自動的に実績値を計算します — 期間、作業量、コスト、日付の実績値と残りの値です。

% Complete #

プロジェクトが進行中の場合、進捗を追跡する必要があります。各タスクの**% Completeを最新の状態に保てば、ルートサマリータスクでプロジェクト全体の% Complete**を確認できます。

特定のタスクの達成率を設定するには、Task Propertiesダイアログの**% Complete**フィールドを使用します。100%完了したタスクは、タスクリストに緑色のチェックマークアイコンが表示されます。

% Completeを更新すると：

  • 0%より上に設定すると、タスクのActual Startがタスクのスケジュール上の開始日に設定されます。
  • 100%に設定すると、タスクのActual Finishがタスクのスケジュール上の終了日に設定されます。
  • Actual DurationとRemaining Durationは達成率に基づいて自動的に計算されます。
  • プロジェクト設定でUpdating task status updates resource statusが有効になっている場合（デフォルト）、Actual WorkとRemaining Workも比例して更新されます。

サマリータスクの**% Complete**は、すべての子孫の非サマリータスクの期間加重平均として計算されます。

Update Projectコマンドを使用して、基準日に基づいて複数のタスクの% Completeを一括で設定することもできます。

Update Project #

Update Projectコマンドは、Projectメニューからアクセスできる一括進捗追跡操作を提供します。

作業完了の更新 #

指定した日付までのタスクを完了としてマークします：

  • 按分（0%〜100%） — 各タスクの稼働期間のうち基準日までにどれだけ経過したかに基づいて、達成率を計算します。
  • 全か無か（0%または100%） — 基準日までに終了するかどうかに基づいて、タスクを0%または100%に設定します。

未完了作業の再スケジュール #

未完了の作業を指定した日付以降に開始されるようにします：

  • まだ開始されていないタスクにはStart No Earlier Than制約が設定されます。
  • 進行中のタスクは、プロジェクトのスケジューリングオプションでSplit in-progress tasksが有効な場合に分割されます。
  • 完了したタスクは変更されません。
