비용 설정 #

작업에 고정 비용을 설정하고 자원에 단가를 설정합니다. Ingantt는 이를 일정과 함께 사용하여 각 작업과 전체 프로젝트의 비용을 계산합니다.

작업 고정 비용 #

Task Properties 대화 상자의 Cost and EV 탭에 있는 Fixed Cost 필드를 채워 자원에 관계없이 작업에 비용을 설정할 수 있습니다. 예를 들어, 누군가가 이미 작업에 대한 총 비용 견적을 제공했거나, 작업에 자원 비용 외의 추가 비용이 있을 때 사용합니다.

여러 작업에 동일한 고정 비용을 배정해야 하는 경우 비용 자원을 만들어 해당 작업들에 배정하는 것을 고려하십시오. 향후 비용이 변경되면 한 곳에서만 업데이트하면 됩니다.

작업 자원 비용 #

Work 자원은 달력이 있는 유일한 유형이므로 비용이 시간, 일, 주 또는 월당으로 지정됩니다(일당 시간, 주당 일수, 월당 일수 참조).

예를 들어, Standard Rate가 시간당 $100인 작업 자원을 Duration이 5시간인 작업에 배정하면 작업의 Cost에 $500이 추가됩니다.

작업 자원은 정규 달력 시간 이외의 작업에 대한 Overtime Rate도 지원합니다. 총 비용은 다음과 같이 계산됩니다:

Cost = Regular Work x Standard Rate + Overtime Work x Overtime Rate + Cost Per Use

작업 자원이 작업에 배정될 때 Units 값을 기본값과 다른 숫자로 지정할 수 있습니다. 이는 비용 계산에 영향을 미칩니다. 예를 들어, Units가 50%이면 작업에서 해당 자원의 계산된 비용은 100% Units일 때의 절반입니다.

자재 자원 비용 #

Material 자원의 경우 단위 정의 방식에 따라 단위당 비용이 지정됩니다. 중량 단위(예: 파운드, 킬로그램, 톤), 길이 또는 부피 단위(예: 피트, 미터, 마일, 갤런, 리터), 기타 단위(예: 컨테이너, 상자, 개, 평방피트)가 될 수 있습니다. Resource Properties 대화 상자에서 Material Label을 설정하여 단위를 설명할 수 있습니다.

자재 자원은 두 가지 소비 모드를 지원합니다:

  • Fixed (기본값) — 작업 기간에 관계없는 총 수량 (예: 시멘트 5톤)
  • Variable — 작업 기간에 비례하는 시간 단위당 비율 (예: 하루 연료 10리터)

예를 들어, 프로젝트에서 연료를 사용하는 경우 "연료" 자재 자원을 추가하고 Resource Properties 대화 상자에서 갤런당 비용을 지정할 수 있습니다. 그런 다음 자원을 작업에 배정할 때 갤런 수를 Units로 지정하면 Ingantt가 연료의 계산된 비용을 작업의 Cost에 추가합니다.

비용 자원 #

Cost 자원은 여러 작업에 배정할 수 있는 고정 비용입니다. 비용 자원은 배정별 비용 값을 지원합니다 — 같은 비용 자원이 다른 작업 배정에서 다른 금액을 가질 수 있습니다(예: "출장" = 작업 A에서 $800, 작업 B에서 $1,200).

이 유형은 설치 비용과 같이 일반적으로 단위로 지정되지 않는 고정 비용에 사용하십시오. 여러 작업이 같은 이유로 동일한 고정 비용을 가지는 경우, 각 작업의 Fixed Cost 필드를 채우는 대신 비용 자원을 만들어 해당 작업들에 배정하십시오.

단가 테이블 #

각 작업 및 자재 자원은 최대 5개의 비용 단가 테이블(A~E)을 지원하며, 각 테이블에는 다른 날짜에 적용되는 여러 행이 있습니다. 이를 통해 시간에 따른 단가 변경을 모델링하거나 다른 유형의 작업에 대해 다른 단가 세트를 유지할 수 있습니다.

단가 테이블은 Resource Properties 대화 상자의 Costs 탭에서 구성됩니다. 자원을 작업에 배정할 때 Edit Resource Assignment 대화 상자에서 사용할 단가 테이블을 선택할 수 있습니다.

사용 당 비용 #

자원은 Cost Per Use 필드를 지원합니다 — 수행된 작업량에 관계없이 자원이 작업에 배정될 때마다 부과되는 정액 요금입니다. 작업 자원의 경우 Cost Per Use는 배정 단위에 곱해집니다(예: 200% 단위는 사용 당 비용이 두 번 부과됨을 의미). Cost Per Use는 항상 작업 시작 시 발생합니다.

비용 발생 #

Cost Accrual 설정은 작업이 진행됨에 따라 비용이 실제 비용으로 인식되는 시점을 제어합니다:

발생 방식 비용 인식 시점
Start 진행이 시작되면 전체 비용 인식 (완료율 > 0%)
Prorated 완료율에 비례 (예: 50% 완료 = 비용의 50%)
End 작업이 100% 완료될 때만 전체 비용 인식

자원에는 Resource Properties 대화 상자의 Costs 탭에 Cost Accrual 설정이 있습니다. 작업에는 Task Properties 대화 상자의 Cost and EV 탭에 Fixed Cost Accrual 설정이 있습니다. 프로젝트에는 Project Properties 대화 상자의 Other 탭에 Default Fixed Cost Accrual 설정이 있습니다.
