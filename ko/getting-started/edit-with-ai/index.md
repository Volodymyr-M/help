# AI로 편집하기

Claude Code, Codex 또는 HTTP 액세스가 가능한 모든 도구와 같은 AI 코딩 도우미에 프로젝트를 연결하여 일정을 실시간으로 변경하도록 합니다. 세션이 진행되는 동안 AI는 현재 프로젝트를 읽고 작업을 추가하거나 수정하고 종속성을 조정하며 자원을 배정하고 업데이트를 다시 보낼 수 있습니다 — 각 변경 사항은 적용되는 순간 Gantt 차트에서 확인할 수 있습니다.

> **AI로 편집하기는 현재 베타 버전입니다.** 이 기능은 Ingantt 웹 버전에서 사용할 수 있습니다.

## AI 편집 세션 시작하기

1. 편집하려는 프로젝트를 엽니다.
2. 오른쪽 상단 구석의 버전 기록 버튼 옆에 있는 **AI로 편집하기** 아이콘(✨)을 클릭합니다.
3. 대화 상자에는 AI 도구가 프로젝트를 어떻게 읽고 써야 하는지 설명하는 지침 세트가 표시됩니다.
4. **지침 복사**를 클릭합니다. 이렇게 하면 편집 세션이 시작되고 지침이 클립보드에 복사됩니다.
5. AI 도구에 지침을 붙여넣습니다 (예: Claude Code 또는 Codex 대화의 첫 번째 메시지로).
6. 자연어로 AI에게 변경 사항을 요청하십시오 — 예를 들어, _"개발 후 2주간의 QA 단계를 추가해 줘"_ 또는 _"Alex를 모든 디자인 작업에 배정해 줘"_.

AI는 지침을 따라 현재 YAML을 가져오고 편집하여 결과를 Ingantt에 다시 보냅니다. Ingantt는 수락하기 전에 모든 편집을 검증합니다.

## 세션 중에 일어나는 일

- **✨ 아이콘 옆의 녹색 점**은 활성 세션을 나타냅니다. 연결이 재연결 중일 때는 점이 황색으로, 세션에 오류가 발생하면 빨간색으로 바뀝니다.
- **모든 AI 변경 사항은** 적용되기 전에 Ingantt의 프로젝트 규칙에 대해 검증됩니다. AI가 유효하지 않은 데이터를 보내면 Ingantt는 변경을 거부하고 AI에게 무엇이 잘못되었는지 알려서 스스로 수정할 수 있도록 합니다.
- **사용자의 편집도 세션에 전달됩니다.** 세션이 활성화된 동안 Ingantt UI에서 무언가를 변경하면 AI는 다음 읽기에서 업데이트를 확인합니다.
- **스케줄러가 최종 권한을 유지합니다.** AI가 날짜나 기간을 설정하면 Ingantt는 임계 경로 방법을 다시 실행하여 최종 일정을 계산하며, 이는 다른 작업을 이동시킬 수 있습니다.

## 세션 중지하기

세션이 활성화된 동안 ✨ 아이콘을 클릭하고 **세션 중지**를 선택합니다. 브라우저 탭을 간단히 닫을 수도 있습니다 — 일정 시간 비활성 상태가 지나면 세션이 자동으로 종료됩니다.

## 지원되는 AI 도구

HTTP 요청을 할 수 있는 모든 AI 도구는 AI로 편집하기와 함께 작동합니다. 일반적인 선택지는 다음과 같습니다:

- **Claude Code** — 대화의 첫 번째 메시지로 지침을 붙여넣습니다.
- **Codex** — 대화의 첫 번째 메시지로 지침을 붙여넣습니다.
- **기타 도구** — `curl` 또는 HTTP 클라이언트에 액세스할 수 있는 도우미는 세션을 읽고 쓸 수 있습니다.

## 팁

- 세션이 활성화된 동안 Ingantt 탭을 열어 두십시오. 탭이 닫히면 AI는 다음 쓰기에서 "세션이 연결되지 않음" 오류를 받게 됩니다.
- AI가 유효하지 않은 YAML을 보내면 Ingantt UI에 알림이 표시됩니다. AI는 오류 세부 정보를 받으며 일반적으로 다음 시도에서 문제를 해결합니다.
- 대규모 프로젝트의 경우 대대적인 변경보다는 집중된 요청(_"테스트 단계 재배치"_)으로 시작하십시오 — AI는 즉각적인 피드백과 함께 작은 단계로 반복할 수 있습니다.
- AI는 제공되지 않은 정보를 볼 수 없습니다. 요청이 문맥 — 제약 조건, 기한 또는 비즈니스 규칙 — 에 의존하는 경우 프롬프트에 포함시키십시오.
