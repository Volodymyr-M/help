# 주공정경로

이 작업들은 절대 지연되어서는 안 됩니다 — 어떤 지연이든 전체 프로젝트를 늦추게 됩니다. 일찍 식별하고, 면밀히 관찰하면 마감일을 지키기 위해 어디에 집중해야 하는지 정확히 알 수 있습니다.

## 주공정 작업

계획을 실행에 옮기면 일부 작업은 예정보다 빨리 끝나고 다른 작업은 그렇지 않습니다. 일부 작업은 프로젝트 타임라인을 연장하지 않고도 더 오래 걸릴 수 있습니다. 이러한 작업에는 _여유_라고 하는 여분의 시간이 있습니다.

다른 작업들은 여유가 전혀 없어 조금이라도 지연되면 프로젝트 종료 날짜가 밀립니다. 이를 _주공정 작업_이라고 합니다. 진행 상황을 추적할 때 이 작업들에 특히 주의를 기울여 프로젝트 일정을 유지하십시오.

다음과 같은 경우에도 작업이 주공정 작업이 됩니다:
- **Must start on** 또는 **Must finish on** [제약 조건](/ko/building-schedule/constraints/index.md#제약-조건)이 있는 경우
- 시작 날짜 기준으로 계획된 프로젝트에서 **As late as possible** [제약 조건](/ko/building-schedule/constraints/index.md#제약-조건)이 있는 경우
- 완료 날짜가 [마감일](/ko/building-schedule/task-properties/index.md#마감일)과 같거나 초과하는 경우
- **음의 여유**가 있는 경우 — 제약 조건이 종속성이 허용하는 것보다 일찍 작업을 강제하는 일정 충돌

100% 완료된 작업은 다른 조건에 관계없이 주공정 작업으로 표시되지 않습니다.

Ingantt는 자동으로 주공정 작업을 감지합니다. **Highlight critical tasks** 옵션이 활성화되어 있으면(**View** 메뉴, 메뉴 바의 **Chart** 메뉴 또는 **Options** 대화 상자를 통해) 이 작업들은 빨간색으로 표시됩니다.

음의 여유가 있는 작업은 작업 목록에 경고 아이콘도 표시되어 일정 충돌을 나타냅니다. 이는 일반적으로 **Start no later than** 또는 **Finish no later than** 제약 조건이 작업의 종속성과 충돌할 때 발생합니다.

## 주공정경로 옵션

**Project Properties** 대화 상자의 **Other** 탭에서 주공정경로 계산 방법을 구성할 수 있습니다:

- **Calculate multiple critical paths** — 활성화하면 연결되지 않은 각 작업 그룹이 자체 주공정경로를 갖습니다. 비활성화(기본값)하면 후속 작업이 없는 작업은 프로젝트 완료 날짜에서 늦은 완료를 도출합니다.
- **Critical slack limit** — 기본적으로 여유가 0이거나 음수인 작업이 주공정 작업입니다. 이 임계값을 높이면 지정된 일수까지의 여유가 있는 작업도 주공정 작업으로 간주됩니다.
