# 자원 평준화

한 사람에게 하루에 감당할 수 있는 것보다 많은 작업을 배정하기 쉽습니다. 평준화는 아무도 과부하되지 않도록 작업을 자동으로 이동시키지만, 프로젝트 완료 날짜가 연장될 수 있습니다.

## 과배정된 자원

작업 자원이 **과배정**될 수 있습니다. 이는 달력과 가용성을 기반으로 완료할 수 있는 것보다 많은 작업이 배정된 것을 의미합니다. 예를 들어, 프로젝트에 두 개의 작업이 있고 각각 기간이 하루이며 서로 [종속성](/ko/building-schedule/dependencies/index.md#선행-작업과-종속성)이 없고, 둘 다 같은 작업 자원에 배정된 경우 해당 자원은 과배정됩니다. 자원이 하루에 이틀분의 작업을 해야 합니다. 이를 해결하려면 두 작업을 종속성으로 연결하면 됩니다.

작업에 과배정된 자원이 배정되어 있으면 Ingantt는 작업 목록에 특수 아이콘을 표시합니다.

자원이 과배정된 경우 Ingantt는 **Resources** 보기와 **Resource Usage** 보기에 특수 아이콘을 표시합니다.

또한 Ingantt는 해당 작업과 자원의 수를 카운트하여 탐색 서랍에 표시합니다.

## 자동 평준화

작업 간의 종속성은 타임라인 상의 위치를 제어합니다. 대규모 프로젝트에서는 종속성을 놓치기 쉬우며, 그 결과 같은 작업 자원에 배정된 여러 작업이 동시에 예약됩니다. 자원이 과배정되고, Ingantt는 작업 및 자원 목록에 특수 아이콘으로 알려줍니다.

종속성이나 제약 조건을 설정하여 일부 작업을 이동시켜 동시에 작업이 수행되지 않도록 수동으로 과배정을 해결할 수 있습니다.

과배정을 해결하는 대안으로 자동 평준화가 있습니다. **Project** 메뉴에서 **Auto-level resources**를 선택하면 Ingantt가 자원 과배정을 방지하기 위해 일부 작업을 타임라인에서 자동으로 뒤로 이동시킵니다. **Project** 메뉴에서 **Clear leveling**을 선택하여 이러한 자동 조정을 지울 수 있습니다.

자동 평준화 명령은 세 가지 범위 옵션을 제공합니다:

- **All tasks** — 프로젝트의 모든 작업을 평준화
- **Selected tasks** — 현재 선택된 작업만 평준화
- **Selected resources** — 선택된 자원의 과배정 충돌만 평준화 (Resources 및 Resource Usage 보기에서 사용 가능)

### 자동 평준화 작동 방식

**Auto-level resources**를 실행하면 Ingantt는 **작업을 지연시키거나 분할**하여 과배정을 해결합니다. 자원을 재배정하거나 배정 단위를 변경하지 않습니다.

알고리즘은 자원에 용량 이상의 작업이 배정된 날을 식별한 다음 경합 작업 중 어느 것을 지연시킬지 결정합니다. 각 작업은 여러 요인을 기반으로 점수가 매겨지며, 점수가 높은 작업은 지연에서 보호됩니다:

| 요인 | 효과 |
|--------|--------|
| **종속성** | 다른 작업의 선행 작업인 작업이 보호됨 |
| **여유** | 주공정 또는 준주공정 작업(여유가 적은)이 보호됨 |
| **우선순위** | 우선순위 값이 높은 작업이 보호됨 |
| **제약 조건** | 일정 제약 조건이 있는 작업이 보호됨 |
| **시작 날짜** | 일찍 시작하는 작업이 어느 정도 보호됨 |
| **기간** | 기간이 긴 작업이 어느 정도 보호됨 |

점수가 가장 낮은 작업이 각 충돌을 해결하기 위해 지연됩니다. 이 과정은 모든 과배정이 해결되거나 더 이상 지연을 적용할 수 없을 때까지 반복됩니다.

> 자동 평준화는 프로젝트 완료 날짜를 연장할 수 있습니다. 가능한 한 많은 과배정을 해결하지만, 제약 조건으로 인해 추가 지연이 불가능한 경우 일부가 남을 수 있습니다.

### 평준화 옵션

**Project** 메뉴에서 접근할 수 있는 **Leveling Options** 대화 상자에서 평준화 동작을 구성합니다:

- **Leveling order** — 작업의 우선순위 결정 방식을 제어: ID Only(ID 오름차순), Standard(다중 요인 점수), Priority Standard(우선순위 먼저, 그다음 표준 요인; 기본값)
- **Overallocation basis** — 감지 세분도 제어: 분, 시간, 일, 주 또는 월. 분 및 시간 설정은 일일 총계가 용량 내에 있더라도 하루 내에 작업이 겹치는 충돌을 감지합니다.
- **Adjust individual assignments** — 활성화되고 작업에 여러 자원 배정이 있는 경우, 전체 작업이 아닌 과배정된 자원의 배정만 지연됩니다.
- **Create splits in remaining work** — 활성화되면 평준화 알고리즘이 전체 작업을 지연하는 대신 충돌 주변으로 작업을 분할할 수 있습니다.
- **Level only within available slack** — 평준화 지연이 작업의 총 여유를 초과하지 않도록 제한하여 프로젝트 완료 날짜 연장을 방지합니다.
- **Leveling range** — 특정 날짜 범위 내의 작업으로 평준화를 제한합니다.

### 평준화 제외 대상

다음 작업 및 자원은 자동 평준화의 영향을 받지 않습니다:

- [우선순위](#우선순위)가 **1000**으로 설정된 작업
- **Can Level**이 **false**로 설정된 작업 (Task Properties 대화 상자의 **Leveling** 탭)
- **Must start on** 또는 **Must finish on** [제약 조건](/ko/building-schedule/constraints/index.md#제약-조건)이 있는 작업
- [요약 작업](/ko/building-schedule/tasks/index.md#요약-작업)
- 기간이 0인 마일스톤
- 완전히 완료된 작업(100%)
- Resource Properties 대화 상자에서 **Can Level**이 **false**로 설정된 자원
- [예약 유형](/ko/building-schedule/resources/index.md#예약-유형)이 **Proposed**인 자원 (제안된 자원을 포함하는 평준화 옵션이 활성화되지 않은 경우)

### 재평준화 및 지우기

**Auto-level resources**를 실행할 때마다 이전 평준화가 먼저 자동으로 지워집니다. 평준화는 항상 깨끗한 비평준화 일정에서 시작하여 지연이 누적되는 것을 방지합니다.

평준화 후 일정을 수정하면(작업 추가, 종속성 변경 등) 기존 평준화 지연은 유지되지만 더 이상 모든 과배정을 해결하지 못할 수 있습니다. **Auto-level resources**를 다시 실행하여 업데이트된 일정을 재평준화하십시오.

모든 평준화 지연을 제거하고 원래 CPM 계산 일정으로 돌아가려면 **Project** 메뉴에서 **Clear leveling**을 선택하십시오.

### 평준화 Gantt

자동 평준화를 실행한 후 Gantt 차트 영역에 **Leveling Gantt** 토글 버튼이 나타납니다. 활성화하면 평준화 전 위치(자동 평준화 전 작업의 위치)에 녹색 막대가 그려지고, 표준 작업 막대는 현재 평준화된 위치에 유지됩니다. 이를 통해 원래 일정과 평준화된 일정을 비교하여 각 작업이 얼마나 지연되었는지 확인할 수 있습니다.

이 토글은 프로젝트에 평준화 데이터가 있을 때만 표시되며 평준화를 지우면 자동으로 숨겨집니다.

## 우선순위

**Priority** 필드는 **Task Properties** 대화 상자의 **Leveling** 탭에서 사용할 수 있습니다. 0에서 1000까지의 정수 값을 입력하며, 기본값은 500입니다.

우선순위는 자원 과배정 해결 시 어떤 작업을 지연시킬지 결정하기 위해 [자동 평준화](#자동-평준화) 알고리즘에서 사용됩니다. 우선순위가 높은 작업은 지연될 가능성이 낮습니다:

- **0** — 가장 낮은 우선순위, 가장 지연되기 쉬움
- **500** — 기본 우선순위
- **1000** — "평준화 안 함" — 자동 평준화에 의해 절대 지연되지 않음

> 외부 마감일이나 고정된 약속에 묶인 작업처럼 자동 평준화에 의해 이동되어서는 안 되는 작업의 경우 Priority를 **1000**으로 설정하십시오.
