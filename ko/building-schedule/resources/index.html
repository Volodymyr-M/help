자원 #

작업을 수행할 인력, 장비 및 자재를 추가합니다. 자원이 작업에 배정되면 Ingantt가 업무량을 계산하고, 과배정을 감지하며, 각 사람의 가용성을 고려할 수 있습니다.

자원 유형 #

Ingantt는 세 가지 유형의 자원을 지원합니다:

자원 유형 설명
Work 작업을 수행하는 사람이나 대상입니다. 개인, 팀, 외주 업체 또는 장비가 될 수 있습니다.
Material 작업을 완료하는 데 사용되는 항목으로, 자재, 재료 또는 부품 등입니다.
Cost 배송 비용, 배포 비용 또는 기타 고정 비용과 같이 여러 작업에 적용할 수 있는 비용입니다.

작업 자원만이 작업에 배정될 때 일정에 영향을 미칩니다.

자원의 Cost 필드가 채워져 있으면 모든 유형의 자원이 비용 계산에 영향을 미칩니다.

최대 단위 #

작업 자원에는 Max Units 속성(기본값 100%)이 있으며, 이는 자원의 사용 가능한 용량을 나타냅니다. 예:

  • 100% — 전일 근무 자원
  • 50% — 반일 근무 자원(시간의 절반만 사용 가능)
  • 300% — 단일 자원으로 표현된 3명의 전일 근무 팀원

Max Units는 자원이 감당할 수 있는 것보다 많은 작업이 있는지 확인하기 위해 과배정 감지 및 자동 평준화에서 사용됩니다. 작업 자원을 작업에 배정할 때 기본 배정 단위는 자원의 Max Units와 일치합니다.

자원 가용 기간 #

작업 자원의 가용성은 시간이 지남에 따라 변경될 수 있습니다. 예를 들어, 팀원이 처음 3개월은 전일 근무하다가 이후 반일 근무로 전환할 수 있습니다.

Resource Properties 대화 상자의 Availability 탭에서 각각 날짜 범위와 가용 비율이 있는 여러 가용 기간을 정의할 수 있습니다. 정의된 기간 내의 날짜에 대해 해당 기간의 단위가 용량 계산 시 자원의 기본 Max Units를 대체합니다.

일반 자원 #

Resource Properties 대화 상자의 체크박스를 사용하여 자원을 Generic으로 표시할 수 있습니다. 일반 자원은 특정 개인이 아닌 역할(예: "개발자", "디자이너")을 나타내는 임시 표시입니다. 특정 팀원이 아직 확정되지 않은 초기 계획 단계에서 유용합니다.

일반 자원은 자원 목록에서 배지 아이콘으로 표시됩니다. 일반 플래그는 정보 제공 목적이며 일정이나 비용 계산에 영향을 미치지 않습니다.

예약 유형 #

각 자원에는 Resource Properties 대화 상자에서 Committed(기본값) 또는 Proposed로 설정할 수 있는 Booking Type이 있습니다.

  • Committed — 자원이 프로젝트에 확정적으로 예약되었습니다.
  • Proposed — 자원이 계획 목적으로 잠정적으로 배정되었습니다.

기본적으로 자동 평준화에서는 제안된 자원이 제외됩니다. 평준화 옵션 대화 상자에서 "Level resources with the proposed booking type"을 활성화하여 포함할 수 있습니다.
