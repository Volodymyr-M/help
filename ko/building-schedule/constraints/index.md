# 제약 조건

일부 작업은 특정 날짜에 시작하거나 완료해야 합니다 — 배송이 화요일에 도착하거나 허가가 금요일에 만료되는 경우 등이 있습니다. 제약 조건을 사용하면 필요한 곳에 날짜를 고정하면서 일정의 나머지 부분은 유연하게 유지할 수 있습니다.

## 제약 조건의 작동 방식

작업 종속성(선행 작업 링크)과 함께 작업의 제약 조건은 작업의 일정 방식을 정의합니다.

제약 조건은 **Task Properties** 대화 상자의 **Advanced** 탭에서 설정됩니다. 기본 제약 조건은 **As soon as possible**입니다. 이는 다른 작업과의 종속성을 고려하여 작업이 프로젝트 시작 날짜에 가능한 한 가깝게 설정됨을 의미합니다. 완료 날짜 기준으로 계획된 프로젝트에서는 기본값이 **As late as possible**입니다.

지정된 날짜에 종속성에 관계없이 작업이 시작되거나 완료되도록 강제하는 두 가지 제약 조건이 있습니다. 이를 _고정 제약 조건_이라 하며, **Must start on**과 **Must finish on**이 해당됩니다. 반드시 필요한 경우에만 이 제약 조건을 사용하십시오.

다른 제약 조건(**Start no earlier than**, **Start no later than**, **Finish no earlier than**, **Finish no later than**)은 작업 종속성을 존중하므로 _유연한 제약 조건_이라고 합니다. 종속성이 작업을 제약 날짜 이후로 밀면 종속성에 의한 날짜가 우선합니다.

| 제약 조건                 | 설명                                                                                                                                   |
|----------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------|
| **As soon as possible**    | 선행 작업이 허용하는 한 가능한 빨리 일정이 잡힙니다. 연결된 선행 작업이 없으면 상위 요약 작업의 시작 시점에 시작됩니다. |
| **As late as possible**    | 선행 작업이 허용하는 한 가능한 늦게 일정이 잡힙니다. 연결된 선행 작업이 없으면 상위 요약 작업의 종료 시점에 완료됩니다.    |
| **Start no earlier than**  | 선행 작업으로 인해 지정된 날짜보다 늦게 시작되면 변경 사항이 없습니다. 그렇지 않으면 지정된 날짜에 시작되도록 일정이 잡힙니다. |
| **Start no later than**    | 선행 작업이 작업을 제약 날짜 이후로 밀면 종속성에 의한 날짜가 우선합니다. 그렇지 않으면 지정된 날짜까지 시작되도록 일정이 잡힙니다. |
| **Finish no earlier than** | 선행 작업으로 인해 지정된 날짜보다 늦게 완료되면 변경 사항이 없습니다. 그렇지 않으면 지정된 날짜에 완료되도록 일정이 잡힙니다.|
| **Finish no later than**   | 선행 작업이 작업을 제약 날짜 이후로 밀면 종속성에 의한 날짜가 우선합니다. 그렇지 않으면 지정된 날짜까지 완료되도록 일정이 잡힙니다. |
| **Must start on**          | 선행 작업에 관계없이 작업의 시작 날짜가 정확히 지정된 대로 예약됩니다.                                                   |
| **Must finish on**         | 선행 작업에 관계없이 작업의 완료 날짜가 정확히 지정된 대로 예약됩니다.                                                  |

유연한 제약 조건 또는 고정 제약 조건이 있는 작업은 작업 목록에 특수 아이콘이 표시됩니다.

> 대부분의 작업은 **As soon as possible**로 유지하고 특정 날짜 근처에서 시작하거나 완료해야 하는 작업에만 유연한 제약 조건을 사용하십시오.
