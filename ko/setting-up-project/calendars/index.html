달력 #

공휴일, 휴가, 반일 근무, 야간 교대 — 일정은 실제로 작업이 수행되는 시간을 반영해야 합니다. 달력을 설정하면 Ingantt가 비근무 시간을 건너뛰고 현실적인 날짜를 제공합니다.

달력은 작업을 수행할 수 있는 날짜와 시간을 정의합니다. 달력은 두 가지 유형의 데이터를 사용합니다:

  • 요일: 요일별로 설정된 근무 시간 — 일반적이고 규칙적인 일정을 정의합니다.
  • 예외: 요일 일정과 다른 시간에 작업이 수행되거나 수행되지 않는 날짜입니다.

프로젝트 달력 #

프로젝트에는 Project Properties 대화 상자에서 지정된 Calendar가 연결되어 있습니다. 이 달력은 작업 자원에 배정되지 않은 작업의 일정 방식을 정의합니다. 주말 등 비근무일은 이 달력을 기반으로 Gantt 차트에도 표시됩니다.

자원 달력 #

각 작업 자원에는 Resource Properties 대화 상자의 Base Calendar 필드에서 지정된 달력이 연결되어 있습니다. 작업 자원이 작업에 배정되면 해당 달력이 작업 일정에 영향을 미칩니다.

여러 작업 자원이 하나의 작업에 배정된 경우, 해당 달력은 교차 처리됩니다 — 즉, 모든 배정된 자원이 가용한 시간에만 작업이 예약됩니다. 예를 들어, 자원 A가 월수 근무하고 자원 B가 수금 근무하면, 두 자원 모두 배정된 작업은 수요일에만 예약됩니다.

여러 자원이 유사한 근무 일정을 공유하는 경우, 하나의 달력을 만들어 모든 자원에 지정하십시오. Resource Properties에서 각 자원별로 별도의 예외(예: 휴가 또는 특정 날짜의 조정된 근무 시간)를 지정할 수 있습니다.

작업 달력 #

각 작업에는 Task Properties 대화 상자에 Calendar 속성이 있으며, 기본적으로 NONE으로 설정되어 있습니다. NONE으로 설정된 경우:

  • 작업에 작업 자원이 배정되지 않은 경우, 프로젝트 달력에 따라 일정이 잡힙니다.
  • 작업에 작업 자원이 배정된 경우, 해당 자원의 달력에 따라 일정이 잡힙니다.

작업의 Calendar 속성이 NONE 대신 특정 달력으로 설정된 경우:

  • 작업에 작업 자원이 배정되지 않은 경우, 지정된 달력에 따라 일정이 잡힙니다.
  • 작업에 작업 자원이 배정된 경우, 작업 달력과 자원 달력의 교차 시간에 따라 일정이 잡힙니다 — 작업 달력과 모든 자원 달력 모두에서 근무 시간인 경우에만 작업이 예약됩니다. 자원 달력을 완전히 무시하고 작업 달력만 사용하려면 Ignore resource calendars 플래그를 체크하십시오.

미리 정의된 달력 #

Ingantt에는 3개의 미리 정의된 달력이 있으며, 그 중 하나(Standard)가 기본적으로 프로젝트에 지정됩니다.

달력 설명
Standard 월요일부터 금요일까지, 오전 8시부터 오후 5시까지 근무하며 1시간(오후 12시~오후 1시) 휴식이 있습니다.
Night Shift 월요일 밤부터 토요일 아침까지, 오후 11시부터 오전 8시까지 근무하며 1시간(오전 3시~오전 4시) 휴식이 있습니다.
24 Hours 매일 24시간 작업이 수행됩니다.

Calendar Properties 대화 상자를 사용하여 미리 정의된 달력을 편집하거나, Add Calendar 대화 상자를 사용하여 이를 기반으로 자체 달력을 만들 수 있습니다.

근무 주간 #

근무 주간을 사용하면 특정 날짜 범위에 대해 주간 일정을 변경할 수 있습니다. 예를 들어, 모든 사람이 표준 8시간 대신 4시간 근무하는 기간이 있거나 토요일이 근무일이 되는 기간을 설정할 수 있습니다.

근무 주간은 Calendar Properties 대화 상자의 Work Weeks 탭에서 구성됩니다. 각 근무 주간에는 이름, 시작 및 종료 날짜, 요일별 근무 시간 정의가 있습니다.

달력 예외는 근무 주간보다 우선하며, 근무 주간은 기본 요일 일정보다 우선합니다.

달력 예외 #

달력 예외는 일반 요일 일정과 다른 작업이 수행되는 날짜이며, 비근무일도 포함됩니다.

달력 예외는 다음에서 지정할 수 있습니다:

  • 특정 자원에 대한 Resource Properties 대화 상자.
  • 이 달력을 사용하는 프로젝트, 자원 또는 작업에 예외를 적용하기 위한 Calendar Properties.

예를 들어, 특정인의 휴가를 구성하려면 해당 자원의 예외로 추가하십시오. 모든 사람에게 공휴일을 구성하려면 모든 사람이 사용하는 달력의 예외로 추가하십시오.

달력 예외를 추가하거나 편집할 때 예외에 대한 근무 시간을 지정합니다. 근무 시간을 지정하지 않으면 해당 예외에는 근무 시간이 없으며, 이는 비근무일을 의미합니다.

각 예외에는 설명용 Name(예: "크리스마스 휴일" 또는 "사무실 휴무")을 지정할 수 있으며, 이는 예외 목록에 표시됩니다.

반복 예외 #

달력 예외는 단일 날짜가 아닌 패턴에 따라 반복되도록 설정할 수 있습니다. 사용 가능한 반복 패턴은 다음과 같습니다:

  • 매일 — N일마다 또는 N 근무일마다
  • 매주 — N주마다 지정된 요일에
  • 매월 — 날짜별 또는 위치별(예: 둘째 화요일)
  • 매년 — 날짜별 또는 위치별(예: 11월 마지막 금요일)

반복 예외에는 종료 조건이 있습니다: 종료 없음, 날짜까지 종료, N회 발생 후 종료.

빈 달력 및 부분 달력 #

달력에 근무 시간을 추가하지 않으면 비어 있어 일정 예약에 사용할 수 없습니다. 이러한 달력은 달력 목록에서 경고 아이콘으로 표시되며, 탐색 서랍에 이러한 달력의 수가 표시됩니다.

그러나 달력이 비어 있지 않지만 특정 작업의 일정을 잡기에 충분한 근무 시간이 없는 경우가 있습니다. 이러한 상황은 일정이 실행될 때만 감지할 수 있습니다.

빈 달력 또는 충분한 근무 시간이 없는 달력을 프로젝트, 자원 또는 작업에 지정하여 일정을 진행할 수 없는 경우, 오류가 표시되고 마지막 작업이 되돌려집니다.
