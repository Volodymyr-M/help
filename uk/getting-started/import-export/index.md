# Імпорт та експорт

Відкривайте файли проектів із багатьох програм планування та експортуйте свій проект у PDF, PNG, CSV або інші формати.

## Імпорт файлів проектів

Ingantt може відкривати файли проектів із багатьох програм планування, не лише з Microsoft Project. Підтримувані формати імпорту включають:

- **Microsoft Project** — MPP, MPT, MPX
- **Primavera P6** — XER, PMXML
- **GanttProject** — GAN
- **Asta Powerproject** — PP
- **ConceptDraw** — CDPX
- **FastTrack Schedule** — FTS
- **ProjectLibre** — POD
- **GNOME Planner** — Planner
- **Phoenix Project Manager** — PPX
- **Synchro** — SP
- **TurboProject** — PEP
- **Ingantt YAML** — YAML, YML
- Та інші (SDEF та інші формати)

Щоб імпортувати файл, натисніть кнопку **+** на екрані проектів або скористайтеся пунктом **Імпорт** у меню **Файл**. Якщо ви вносите зміни до імпортованого файлу, Ingantt зберігає їх у форматі XML.

> Для відкриття файлів проектів, відмінних від XML, Ingantt надсилає їх через безпечне з'єднання до веб-служби Ingantt для конвертації. Ваші файли не зберігаються на сервісі. Потрібне підключення до інтернету. Файли YAML обробляються локально.

## Експорт вашого проекту

Ingantt надає кілька форматів експорту через пункт **Експорт** у меню **Файл** (або **Завантажити** у веб-версії). Коли ви обираєте **Експорт**, з'являється діалог, що дозволяє вибрати потрібний формат:

| Формат | Опис |
|--------|------|
| **PDF** | Документ PDF для друку з вашою діаграмою Ганта |
| **PNG** | Файл зображення вашої діаграми Ганта (включає поточну тему) |
| **CSV** | Файл зі значеннями, розділеними комами, з даними завдань для електронних таблиць |
| **XML** | Формат XML проекту для сумісності з іншими програмами |
| **YAML** | Компактне, зручне для людини визначення проекту, створене для редагування за допомогою ШІ |
| **Markdown** | Текстовий формат для документації, README або вікі |

Експортований файл відображає поточні налаштування вашої діаграми Ганта. Наприклад, якщо назви завдань приховані в діалозі **Параметри**, вони також не з'являться в експорті. Масштаб також зберігається.

> Експорт у PDF завжди використовує кольори світлої теми, тоді як експорт у PNG використовує поточну тему інтерфейсу (світлу або темну). Експорт у CSV, YAML та Markdown — це звичайний текст. Коли вони видимі в інтерфейсі, розділені завдання, базові стовпці та стовпці вирівнювання відображаються в експорті PDF та PNG.

## Редагування YAML за допомогою ШІ-агентів

Експорт у YAML створений для редагування за допомогою ШІ-асистента програмування (Claude Code, Codex або будь-якого інструмента, що вміє читати й записувати файли). Експортуйте проект, опишіть потрібні зміни природною мовою — _«додай двотижневу фазу QA після розробки»_, _«признач Alex на всі завдання з дизайну»_ — та імпортуйте результат назад.

Формат містить **лише вхідні дані** (тривалості, залежності, обмеження, призначення, календарі, ставки). Похідні значення — дати початку/завершення завдань, загальна вартість, критичний шлях, резерв часу, підсумки зведень — навмисно опускаються; Ingantt перераховує їх під час імпорту. Агент не повинен намагатися обчислювати дати самостійно.

Направте агента до [довідника формату YAML](/yaml-reference.md), щоб він знав схему.

> Для редагування в реальному часі без циклу експорт/імпорт див. [Редагування за допомогою ШІ](/uk/getting-started/edit-with-ai/index.md) — ШІ читає й записує ту саму YAML-схему через живу сесію, і ви бачите кожну зміну на діаграмі Ганта в момент її появи.
