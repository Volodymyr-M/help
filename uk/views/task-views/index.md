# Подання завдань

Окрім діаграми Ганта, переглядайте завдання на дошці Канбан, місячному календарі, мережевій діаграмі або часовій шкалі. Кожне подання висвітлює різні аспекти вашого розкладу.

## Дошка завдань

Подання **Дошка завдань** відображає завдання як картки, організовані у стовпці, що представляють етапи робочого процесу (стиль Канбан).

Нові проекти починаються з трьох стовпців за замовчуванням: «Не розпочато» (0%), «В процесі» (50%) та «Завершено» (100%). Кожен стовпець має відповідність % виконання — коли ви перетягуєте картку до стовпця, відсоток виконання завдання автоматично оновлюється.

Ви можете додавати, редагувати, перейменовувати, переміщувати та видаляти стовпці. Кожен стовпець має кнопку **Додати завдання** внизу для створення завдань безпосередньо на цьому етапі.

На дошці відображаються лише несумарні, активні завдання з увімкненим параметром «Показувати на дошці». Параметр «Показувати на дошці» доступний на вкладці **Візуальне** діалогу **Властивості завдання**.

> Перетягуйте картки завдань між стовпцями, щоб змінити їхній статус. Коли % виконання завдання змінюється іншим способом (через діалог, редагування в таблиці або «Оновити проект»), картка автоматично переміщується до відповідного стовпця.

## Мережева діаграма

Подання **Мережева діаграма** відображає завдання як прямокутні вузли, розташовані у вигляді блок-схеми зліва направо. Завдання без попередників з'являються в крайньому лівому стовпці, а кожне наступне завдання — після всіх своїх попередників.

- Звичайні завдання показують ID, тривалість, дати початку/завершення та ресурси
- Сумарні завдання показують ID, тривалість, дати початку/завершення та відсоток виконання
- Віхи показують ID та дату початку
- Завдання критичного шляху мають червоні заголовки та межі
- Прогрес позначається діагональними лініями: одна лінія для завдань у процесі, перехресні лінії для завершених

Ви можете згортати/розгортати сумарні завдання, клацати для вибору, двічі клацати для редагування та клацати правою кнопкою для контекстного меню. Стрілки залежностей малюються між вузлами для всіх чотирьох типів зв'язків.

## Календарне подання

**Календарне подання** відображає місячну сітку календаря із завданнями у вигляді горизонтальних смуг, що охоплюють їхні заплановані дати.

- Переключайтесь між поданнями **Місяць** та **Тиждень** за допомогою перемикача в заголовку
- Увімкніть режим **Робочий тиждень**, щоб показувати лише робочі дні (5-денний вигляд)
- Переміщуйтесь кнопками «Попередній»/«Наступний» або перейдіть до сьогодні
- Неробочі дні показані з відмінним фоном
- Віхи відображаються як іконки ромбів
- Двічі клацніть на порожньому дні, щоб створити нове 1-денне завдання на цю дату

Завдання, що перетинають межу тижня, переносяться на наступний рядок. Коли кілька завдань перетинаються в один день, вони розташовуються вертикально.

## Часова шкала

**Часова шкала** — це панель, що згортається, та з'являється над діаграмою Ганта, показуючи спрощений огляд вибраних користувачем завдань на горизонтальній осі часу.

Щоб додати завдання на часову шкалу, клацніть на ньому правою кнопкою миші та оберіть **Відображати на часовій шкалі**, або позначте цей параметр на вкладці **Візуальне** діалогу «Властивості завдання».

Завдання на часовій шкалі можуть відображатися як смуги або виноски (текстові мітки з вертикальними з'єднувальними лініями). Ви можете перемикати між режимами, перетягуючи смугу завдання вище або нижче горизонтальної розділової лінії на панелі часової шкали. Часова шкала також показує маркер «сьогодні» та накладку області перегляду, яку можна перетягувати для прокручування діаграми Ганта.

Перемикайте часову шкалу з меню **Подання**.

## Діаграма Ганта для відстеження

Подання **Діаграма Ганта для відстеження** подібне до стандартного подання завдань, але з увімкненим виділенням критичного шляху та відображенням Базового плану 0, якщо він був збережений. Коли ви переходите з подання Діаграми Ганта для відстеження, налаштування видимості критичного шляху та базового плану відновлюються до попереднього стану.

Використовуйте Діаграму Ганта для відстеження для моніторингу прогресу проекту порівняно з базовим планом.
