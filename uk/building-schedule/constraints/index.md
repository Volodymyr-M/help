# Обмеження

Деякі завдання повинні починатися або завершуватися в конкретні дати — доставка приходить у вівторок, дозвіл закінчується в п'ятницю. Обмеження дозволяють зафіксувати дати там, де потрібно, зберігаючи гнучкість решти розкладу.

## Як працюють обмеження

Разом із залежностями завдань (зв'язками з попередниками), обмеження завдання визначає, як воно планується.

Обмеження встановлюються на вкладці **Додатково** діалогу **Властивості завдання**. Стандартне обмеження — **Якомога раніше**. Це означає, що завдання встановлюється якомога ближче до дати початку проекту з урахуванням залежностей з іншими завданнями. У проектах, що плануються від дати завершення, стандартне обмеження — **Якомога пізніше**.

Є два обмеження, які примушують завдання починатися або завершуватися у зазначену дату незалежно від залежностей. Ці обмеження називаються _негнучкими_ і є **Повинно початись** та **Повинно завершитись**. Використовуйте ці обмеження лише якщо ви впевнені в їх необхідності.

Інші обмеження (**Почати не раніше ніж**, **Почати не пізніше ніж**, **Завершити не раніше ніж** та **Завершити не пізніше ніж**) називаються _гнучкими_, оскільки вони враховують залежності завдань. Якщо залежності зсувають завдання за межі дати обмеження, дата, визначена залежністю, має пріоритет.

| Обмеження | Опис |
|-----------|------|
| **Якомога раніше** | Завдання планується якомога раніше, як дозволяють попередники. Якщо попередників немає, завдання починається на початку батьківського сумарного завдання. |
| **Якомога пізніше** | Завдання планується якомога пізніше, як дозволяють попередники. Якщо попередників немає, завдання завершується наприкінці батьківського сумарного завдання. |
| **Почати не раніше ніж** | Якщо завдання починається пізніше за зазначену дату через попередників, нічого не змінюється. В іншому випадку завдання планується на зазначену дату. |
| **Почати не пізніше ніж** | Якщо попередники зсувають завдання за дату обмеження, дата залежності має пріоритет. В іншому випадку завдання планується на зазначену дату. |
| **Завершити не раніше ніж** | Якщо завдання завершується пізніше за зазначену дату через попередників, нічого не змінюється. В іншому випадку завдання планується на зазначену дату завершення. |
| **Завершити не пізніше ніж** | Якщо попередники зсувають завдання за дату обмеження, дата залежності має пріоритет. В іншому випадку завдання планується на зазначену дату завершення. |
| **Повинно початись** | Дата початку завдання встановлюється точно так, як зазначено, незалежно від попередників. |
| **Повинно завершитись** | Дата завершення завдання встановлюється точно так, як зазначено, незалежно від попередників. |

Завдання з гнучким або негнучким обмеженням відображають спеціальну іконку у списку завдань.

> Залишайте більшість завдань з обмеженням **Якомога раніше** та використовуйте гнучкі обмеження лише для завдань, які повинні починатися або завершуватися близько до певної дати.
